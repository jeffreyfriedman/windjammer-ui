‚ú® What's New in v0.3.0

Complete API redesign with type-safe components, structured styling, and zero HTML/CSS strings. All components now use the Component enum and Style builder.

üéØ Type-Safe Children

No more raw HTML strings! All children are now typed as Component.

// Before (v0.2.x) Container::new() .child("<div>Hello</div>") // After (v0.3.0) use windjammer_ui::html::Div Container::new() .child(Div::new().render())

üé® Structured Styling

Type-safe CSS properties with autocomplete and validation.

// Before (v0.2.x) .style("padding: 16px") // After (v0.3.0) use windjammer_ui::core::Style .style(Style::new() .padding("16px") .background_color("blue"))

üèóÔ∏è HTML Elements as Structs

Native HTML elements like Div, Span, P are now Windjammer structs.

use windjammer_ui::html::{Div, P, Span} Div::new() .child(P::new() .child(Text::new("Hello")) .render()) .render()

üîÑ Migration Guide

How to upgrade from v0.2.x to v0.3.0

Step 1: Update Imports

// Add new imports for type-safe API use windjammer_ui::core::{Component, Style} use windjammer_ui::html::{Div, P, Span}

Step 2: Convert Children

// Before Container::new() .child("<p>Hello</p>".to_string()) // After Container::new() .child( P::new() .child(Text::new("Hello").render()) .render() )

Step 3: Convert Styles

// Before .style("padding: 16px; background: blue".to_string()) // After .style( Style::new() .padding("16px") .background_color("blue") )
