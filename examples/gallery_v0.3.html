How the trait-based system works

// The Renderable trait (defined in traits.wj):
pub trait Renderable {
    fn render(self) -> string
}

// Every component implements it:
impl Renderable for Button {
    fn render(self) -> string {
        format!("<button>{}</button>", self.label)
    }
}

// Generic child() methods accept any Renderable:
impl Div {
    pub fn child<T: Renderable>(self, component: T) -> Div {
        // Automatically calls .render() internally
        self.children.push(component.render())
        self
    }
}
  • Trait Bounds: Generic parameters ensure type safety
  • Automatic Rendering: .render() called internally by parent
  • Composability: Any Renderable can be a child of any container
  • Compiler Verified: Type errors caught at compile time

üìñ Migration Guide

Upgrading from v0.2.x to v0.3.0

Step 1: Update Imports

// Add Renderable to your imports:
use windjammer_ui.components.generated.{
    Renderable, // ‚Üê Add this
    Button, Input, Div, Text
}

Step 2: Remove .render() Calls

// Before:
Div::new()
    .child(Text::new("Hello").render())

// After:
Div::new()
    .child(Text::new("Hello"))

Step 3: Keep Root .render()

// Only the root component needs .render():
let page = Div::new()
    .child(Text::new("Hello"))

println!("{}", page.render()) // ‚Üê Keep this
